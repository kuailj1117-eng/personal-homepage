My story

I build software across the full stack: from Python computer vision and Pygame game development, to PHP REST APIs with vanilla JavaScript SPAs, to Node.js tooling and MCP server architecture. My focus is on creating tools that feel polished and interfaces that respond with intent.

What drives me is the intersection of AI and developer tooling. I believe the future of software development is AI-augmented, and I spend my time building Claude Code skills, MCP servers, and automation workflows that turn AI from a chatbot into a productive member of the engineering team.

What I value

Code should be simple

Complexity is a liability. The best solutions are the ones you can explain in one breath. I reach for frameworks only when vanilla solutions stop scaling.

Interfaces should breathe

Every animation needs a reason. Motion communicates hierarchy, state change, and feedback. Dead interfaces feel broken; over-animated ones feel drunk.

AI is a tool, not a replacement

I use AI to amplify my craft, not outsource it. The best AI-assisted code is the kind where you can explain every line without looking at the commit history.

Ship for real people

Dark mode, reduced motion, keyboard navigation, readable contrast. These are not nice-to-haves. If your interface breaks for 10% of users, it's broken.
